test-cli
설치 기능 사용법 리포트 예시 GitHub

여섯 개 언어, 하나의 표준 리포트.

test-cli는 프로젝트의 언어를 자동 감지해 테스트를 커버리지와 함께 실행하고, 언어와 무관한 단일 스키마로 정규화합니다. 기계 판독용(JSON·JUnit·Cobertura)부터 사람용(Markdown·HTML 대시보드·code-cov 스타일 커버리지 히트맵)까지 한 번에 만듭니다.

설치하기 라이브 리포트 보기 → GitHub Action 
# 최신 릴리스를 설치 (Linux/macOS)
curl -fsSL https://jhl-labs.github.io/test-cli/install.sh | bash

# 버전 고정
curl -fsSL https://jhl-labs.github.io/test-cli/install.sh | VERSION=v0.1.1 bash

왜 test-cli 인가

표준화된 스키마

모든 언어를 test-cli/report@1 JSON 하나로 정규화 — 파이프라인과 AI 에이전트가 출력 하나만 읽으면 됩니다.

커버리지 히트맵

code-cov 수준의 HTML 대시보드와 줄 단위 hit/miss 히트맵을 자동 생성합니다.

내용 기반 파싱

JUnit·go test -json·Cobertura·LCOV·JaCoCo·Go coverprofile을 파일명이 아닌 내용으로 감지합니다.

안정적 종료 코드

0 통과 · 1 실패/커버리지 게이트 · 2 사용법 · 3 실행실패 · 4 환경 — CI 계약으로 고정.

AI 에이전트 스킬

generate-skill로 에이전트가 바로 쓰는 SKILL.md를 생성합니다.

GitHub Action

test-cli-action으로 설치·실행·어노테이션·Job Summary를 한 번에.

지원 언어 & 포맷

Python · pytest → JUnit + Cobertura TypeScript · jest → JUnit + Cobertura/LCOV Go · go test -json + coverprofile Rust · cargo-llvm-cov → Cobertura C# · dotnet test → JUnit + Cobertura Java · surefire + JaCoCo

빠른 시작

  1. test-cli doctor — 감지된 언어의 툴체인이 준비됐는지 확인 (없으면 종료 코드 4).
  2. test-cli run . --profile ci --output-dir reports/test — 테스트+커버리지 실행, 리포트 생성.
  3. reports/test/report.json 을 읽어 결과 판단 (test.summary.failed == 0 이면 통과).
  4. reports/test/index.html 와 coverage/index.html 로 대시보드·히트맵 확인.
  5. 커버리지 게이트: test-cli run . --fail-under 80.
